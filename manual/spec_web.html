x-ukagaka-link scheme

[Currently SSP only] A URL scheme called x-ukagaka-link is available to perform something from a website with a single click.
The character encoding must always be UTF-8 and the string must be URL-encoded.
Here is how to write it.

x-ukagaka-link:type=event&ghost=(Ghost Name)&info=(Additional information)
When the URL is opened, the OnXUkagakaLinkOpen event is fired.
(Ghost name) is the same as name or sakura.name in descript.txt, and (Additional information) is an arbitrary string.
Sample: x-ukagaka-link:type=event&ghost=Emily&info=sweet-potato-feeding
x-ukagaka-link:type=install&url=(Encoded URL)
When the URL is opened, the NAR file at the specified URL is installed.
(Encoded URL) is the URL-encoded NAR location.
Sample:x-ukagaka-link:type=install&url=http%3A%2F%2Fssp.shillest.net%2Fghost%2Fbonsyou.nar
x-ukagaka-link:type=homeurl&url=(Encoded URL)
[2.5.76] When the URL is opened, it recognizes the specified URL as an update destination URL. It will obtain a group of files in the same way as sa network update, and then intalls them.
(Encoded URL) is the URL-encoded NAR location.
Sample:x-ukagaka-link:type=homeurl&url=http%3A%2F%2Fms.shillest.net%2Fghost%2Fghost%2Fmobilemaster%2F

application/x-nar MIME type

[Currently SSP only] The MIME type of application/x-nar is recommended for NAR files.
However, application/zip, etc., are also possible.
